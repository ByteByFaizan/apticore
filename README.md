## About

**AptiCore** is an AI-powered recruitment platform that eliminates unconscious bias from the hiring process. It anonymizes candidate data, evaluates skills using Google's Gemini AI, and provides transparent, explainable rankings вҖ” ensuring every candidate is judged on **merit alone**.

Built for the **Google Solution Challenge 2026**, AptiCore demonstrates how AI can be used responsibly to promote fairness and equity in employment.

---

## The Problem

Traditional hiring processes are rife with unconscious bias:

- **Name bias** вҖ” Candidates with "foreign-sounding" names receive 30% fewer callbacks
- **Gender bias** вҖ” Identical resumes with male names are rated higher than those with female names
- **Institution bias** вҖ” Ivy League graduates are preferred regardless of actual skill fit
- **Ethnicity bias** вҖ” Racial minorities face systematic disadvantages in resume screening

AptiCore solves this by **removing all personally identifiable information (PII) before evaluation** and using AI to assess candidates purely on their skills, experience, and job-requirement alignment.

---

## Key Features

### Blind Evaluation
Strips names, emails, phone numbers, photos, addresses, and institution names before any AI analysis begins. Candidates become `Candidate Alpha-1`, `Beta-3`, etc.

### AI-Powered Skill Matching
Uses a **hybrid matching engine** combining keyword extraction with semantic similarity (via Gemini embeddings) to evaluate how well each candidate's skills align with the job description.

### Bias Detection & Reporting
Runs a **dual-pipeline comparison** вҖ” simulating how a biased traditional system *would have* ranked candidates vs. how AptiCore's fair system ranks them вҖ” producing quantified fairness scores with before/after deltas.

### Explainable AI
Every ranking comes with a plain-English explanation generated by Gemini, detailing *why* a candidate scored the way they did. No black boxes.

### Multi-Format Resume Parsing
Accepts **PDF**, **DOCX**, and **TXT** resumes. Extracts structured data (skills, experience, education) using Gemini's document understanding capabilities.

### Enterprise-Grade Security
Multi-layer defense: Edge CORS, CSRF protection, token-bucket rate limiting, Zod input validation, and strict Firestore/Storage security rules.

## Getting Started

### Prerequisites

- **Node.js** >= 18.17
- **npm** >= 9.0
- A [Firebase project](https://console.firebase.google.com/) with Auth, Firestore, and Storage enabled
- A [Google AI Studio](https://aistudio.google.com/) API key for Gemini

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/ByteByFaizan/apticore.git
cd apticore

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.example .env.local
# Edit .env.local with your credentials (see below)

# 4. Start the development server
npm run dev
```

The app will be available at `http://localhost:3000`.

### Available Scripts

| Command | Description |
|---------|-------------|
| `npm run dev` | Start development server with hot reload |
| `npm run build` | Create optimized production build |
| `npm run start` | Start production server |
| `npm run lint` | Run ESLint checks |

## Environment Variables

Create a `.env.local` file in the root directory. See [`.env.example`](.env.example) for the template.

### Required Variables

| Variable | Description | Source |
|----------|-------------|--------|
| `NEXT_PUBLIC_FIREBASE_API_KEY` | Firebase client API key | Firebase Console вҶ’ Project Settings |
| `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN` | Auth domain | Firebase Console вҶ’ Project Settings |
| `NEXT_PUBLIC_FIREBASE_PROJECT_ID` | Project ID | Firebase Console вҶ’ Project Settings |
| `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET` | Storage bucket URL | Firebase Console вҶ’ Project Settings |
| `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID` | Messaging sender ID | Firebase Console вҶ’ Project Settings |
| `NEXT_PUBLIC_FIREBASE_APP_ID` | App ID | Firebase Console вҶ’ Project Settings |
| `GOOGLE_GEMINI_API_KEY` | Gemini API key | [Google AI Studio](https://aistudio.google.com/) |
| `FIREBASE_PROJECT_ID` | Admin SDK project ID | Firebase Console вҶ’ Service Accounts |
| `FIREBASE_CLIENT_EMAIL` | Admin SDK client email | Firebase Service Account JSON |
| `FIREBASE_PRIVATE_KEY` | Admin SDK private key | Firebase Service Account JSON |

### Optional Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `CORS_ALLOWED_ORIGINS` | `""` | Comma-separated production origins |
| `RATE_LIMIT_CREATE` | `10` | Batch creation limit (req/min) |
| `RATE_LIMIT_UPLOAD` | `100` | Upload limit (req/min) |
| `RATE_LIMIT_PROCESS` | `5` | Pipeline trigger limit (req/min) |
| `RATE_LIMIT_READ` | `60` | Read operations limit (req/min) |
| `RATE_LIMIT_WRITE` | `30` | Write operations limit (req/min) |
| `RATE_LIMIT_AUTH` | `5` | Auth operations limit (req/min) |

> **Important:** Never prefix server-side secrets with `NEXT_PUBLIC_`. The `GOOGLE_GEMINI_API_KEY` and `FIREBASE_*` admin credentials must remain server-only.

## AI Pipeline

When a batch is processed, it flows through a **9-step automated pipeline** using Next.js `after()` for background execution:

```
Step 1   CREATED         вҶ’ Batch initialized
Step 2   UPLOADING       вҶ’ Resumes stored in Cloud Storage
Step 3   PARSING         вҶ’ Gemini 2.5 Pro extracts structured data from resumes
Step 4   ANONYMIZING     вҶ’ PII stripped (names, emails, phones, institutions)
Step 5   EXTRACTING_JD   вҶ’ Gemini 2.5 Pro extracts skills & requirements from JD
Step 6   MATCHING        вҶ’ Hybrid engine scores candidates against JD
Step 7   BIAS_BEFORE     вҶ’ Simulated traditional (biased) hiring pipeline
Step 8   BIAS_AFTER      вҶ’ Fair pipeline comparison & fairness scoring
Step 9   COMPLETE        вҶ’ Results ready with rankings & explanations
```

### AI Models Used

| Model | Purpose | Configuration |
|-------|---------|---------------|
| **Gemini 2.5 Pro** | Resume parsing, JD extraction | `temperature: 0`, structured JSON output |
| **Gemini 2.5 Flash** | Candidate explanations | `temperature: 0.3`, natural language output |
| **gemini-embedding-001** | Semantic skill similarity | 768-dimensional vectors |

### Matching Strategy

The hybrid matcher combines two approaches:

1. **Keyword Matching** вҖ” O(1) skill overlap via normalized keyword sets
2. **Semantic Similarity** вҖ” Cosine similarity between Gemini embeddings of candidate skills and JD requirements

Final score = `(keywordScore * 0.6) + (semanticScore * 0.4)` вҖ” weighted toward keyword precision with semantic understanding as a boost.

## Security

AptiCore implements defense-in-depth with multiple security layers:

| Layer | Mechanism | Details |
|-------|-----------|---------|
| **Edge** | CORS Middleware | Origin allowlist enforced at edge; blocks unauthorized origins |
| **Transport** | CSRF Protection | `X-Requested-With` header required on all mutations |
| **Rate Limiting** | Token Bucket | Per-IP/user limits with RFC-compliant `RateLimit-*` headers |
| **Validation** | Zod Schemas | All request bodies validated against strict schemas |
| **Auth** | Firebase Auth | JWT verification on every protected route |
| **Database** | Firestore Rules | Owner-based access; server-only writes for processing data |
| **Storage** | Storage Rules | File type restrictions (PDF/DOCX/TXT), 5MB max, owner-only access |
| **Headers** | Security Headers | CSP, HSTS, X-Frame-Options, X-Content-Type-Options via `next.config.ts` |
